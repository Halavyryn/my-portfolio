Артефакты системного аналитика

Документация требований

Документация требований — ключевой артефакт, отражающий потребности бизнеса и пользователей. Включает user stories, use cases и сценарии взаимодействия. 

Как [Пользователь личного кабинета],
Я хочу [возможность сбросить пароль через email],
Чтобы [восстановить доступ при утере пароля].

Acceptance Criteria:

1. Кнопка "Забыли пароль?" на странице входа
2. Поле для ввода email с валидацией формата
3. Отправка письма с токеном (TTL 24 часа)
4. Страница ввода нового пароля при переходе по ссылке
5. Валидация пароля (мин. 8 символов, цифра, спецсимвол)
6. Уведомление об успешном изменении
